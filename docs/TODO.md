# 待办 / 延后事项

## 监测维度补充（模拟机受限，需真机验证）

### 音量键检测
- **目标**：监测期间按音量加/减键记为一次"操作"。
- **尝试过的方案**：在 `MonitorPage` 根容器加 `.focusable(true).defaultFocus(true)` + `onKeyEvent`，匹配 `KeyCode.KEYCODE_VOLUME_UP / KEYCODE_VOLUME_DOWN`，命中即 `recordScreenLeave()`。
- **结果**：模拟机上 `onKeyEvent` 未触发（系统层可能在到达组件焦点系统前就拦截了音量键）。已回退，避免界面文案承诺未生效的功能。
- **后续可试**：① 真机上重测同一方案；② 改用 UIAbility 级按键监听 / `inputConsumer` 系统按键订阅（可能需要权限）。

### 截屏检测
- **目标**：监测期间截屏记为一次"操作"。
- **结果**：模拟机对截屏不派发任何事件；HarmonyOS 出于隐私对截屏监听本就受限且不可靠。
- **后续可试**：真机上验证是否有可用的系统截屏事件 API；若无，放弃此维度。

> 现有"操作"计数已覆盖：亮屏/息屏广播、离开监测页（返回、回桌面、切应用、下拉通知栏/控制中心失焦）。音量键、截屏属于增强项，非核心。
